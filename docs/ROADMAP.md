# 唤我（HuanWo）开发路线图

**版本：** v0.2  
**日期：** 2026-06-07  
**状态：** 规划中

---

## 总体策略

采用「先验证、后铺代码」的策略：

1. **Phase 0 Spike（技术验证）**：先跑通最小验证项目，确认 sharp 打包、快捷键注册、托盘等关键技术风险点，**不过这关不写业务代码**
2. **Phase 0**：搭建可运行的最小框架（空壳）
3. **Phase 1**：集成第一批工具，形成可用产品
4. **Phase 2**：完善系统功能，重构为插件架构
5. **Phase 3**：扩展工具库，启动 macOS 适配

> **关于插件系统**：Phase 0-1 工具硬编码，不抽象插件系统。Phase 2 再重构为插件架构。详见 [ARCHITECTURE.md](./ARCHITECTURE.md) 第 4 节。

---

## Phase 0 Spike：技术风险验证（必须在正式开发前完成）

> **这是 Phase 0 之前必须完成的验证阶段，建议 1–2 天完成。**
> 详见 [PHASE0_SPIKE.md](./PHASE0_SPIKE.md)。

**必须验证通过的风险项**：

- [ ] `sharp` 在 electron-builder 打包后能否正常运行（最关键）
- [ ] Electron 全局快捷键在 Windows 各版本能否正常注册
- [ ] 系统托盘图标在深浅色任务栏均能正常显示
- [ ] `nativeTheme` 深色/浅色切换实时生效
- [ ] 构建出的安装包能在干净 Windows 虚拟机中正常运行
- [ ] （可选）Tauri 等效方案验证，作为技术选型最终确认依据

**验证方式**：创建一个最小 spike 项目（不写任何业务代码），只验证上述 5 项。

**决策门**：上述必须项全部验证通过后，方可进入 Phase 0 正式开发。

---

## Phase 0：框架搭建（Skeleton）

**前置条件**：Phase 0 Spike 验证全部通过

**目标**：应用可以启动、托盘驻留、快捷键呼出空面板、关闭面板

**任务清单**

- [ ] 初始化 electron-vite + React + TypeScript 项目（按 Spike 验证结果选择 Electron 或 Tauri）
- [ ] 配置 electron-builder（Windows NSIS）
- [ ] 实现系统托盘驻留（带右键菜单）
- [ ] 实现全局快捷键（`Alt+Space`）注册
- [ ] 实现快捷面板窗口（无边框、透明背景、居中显示）
- [ ] 实现面板动画（弹出/关闭）
- [ ] 实现 `Esc` 关闭面板
- [ ] 集成 `nativeTheme` 深色/浅色模式切换
- [ ] 实现 `electron-store` 配置持久化
- [ ] 搭建工具窗口容器（Shell）
- [ ] 初始化本地日志方案（按 [LOGGING.md](./LOGGING.md) 配置 electron-log）

**验收标准**：能构建成 Windows 安装包，安装后可正常启动，快捷键可呼出面板

## Phase 1：第一批工具上线（v0.1 Alpha）

**目标**：图片压缩 + 图片转格式两款工具完整可用，含剪贴板集成

**任务清单**

- [ ] 图片压缩工具 UI（拖放区 + 参数配置 + 文件列表）
- [ ] 图片压缩处理逻辑（sharp，支持 JPEG/PNG/WebP/GIF/AVIF）
- [ ] 图片转格式工具 UI
- [ ] 图片转格式处理逻辑（sharp，主流格式互转，GIF→WebP/APNG 保留动画）
- [ ] 文件拖拽支持（拖入工具窗口）
- [ ] 文件选择器（多选）
- [ ] 剪贴板作为输入源（检测剪贴板图片，自动填入）
- [ ] 剪贴板去重（`onFocus` 时检测到相同图片或未变化内容则跳过，不重复追加）
- [ ] 剪贴板临时文件清理（窗口关闭时 + 应用退出时 + 启动时清理残留，详见 TOOLS_SPEC）
- [ ] 剪贴板输出（处理完成后复制到剪贴板）
- [ ] 处理进度实时展示（每文件进度条）
- [ ] 处理完成摘要 + 打开输出目录
- [ ] 工具面板搜索（关键词过滤）
- [ ] 工具固定/取消固定（持久化）
- [ ] 数字快捷键（1–9 打开固定工具）
- [ ] 错误处理（格式不支持、权限不足、磁盘不足）
- [ ] 本地日志覆盖工具处理路径（便于排查用户问题）
- [ ] **工具处理逻辑单元测试**（image-compress + image-convert 核心处理函数，最小覆盖范围：
  - 格式转换正确性：JPEG→WebP、PNG→WebP、GIF→WebP（含动画帧数验证）
  - 尺寸缩放计算：等比缩放结果验证（含边界值）
  - 边界条件：0 字节文件、超大文件 >500MB、不支持格式（如 .exe）
  - 压缩后体积反增时的原文件保留逻辑）

**验收标准**：两款工具均可完整走通处理流程；剪贴板输入/输出流程顺畅；错误场景有明确提示

## Phase 2：系统完善 + 插件重构（v0.2 Beta）

**目标**：产品具备分发给他人的完整质量

**任务清单**

- [ ] 设置中心（快捷键自定义 + 开机自启 + 关于）
- [ ] 自动更新（`electron-updater` + GitHub Releases + Gitee 镜像）
- [ ] 更新提示 UI（托盘气泡 + 设置页提示）
- [ ] 快捷键冲突检测与提示
- [ ] 资源管理器右键菜单集成（Windows Shell 扩展，右键文件 → "唤我 → 处理"）
- [ ] 处理参数历史记忆（记住最近使用的参数，下次自动恢复）
- [ ] 添加文件批量重命名工具
- [ ] **重构为插件架构**（将 Phase 1 硬编码工具抽象为插件，实现 PluginRegistry）
- [ ] 安装包优化（ASAR 压缩，减少体积）
- [ ] 错误日志导出功能（设置页"导出日志"）
- [ ] 内测分发包（含 Release Notes）
- [ ] **购买 EV 代码签名证书**（v1.0 前必须完成）

**验收标准**：安装包 < 80MB；自动更新流程完整可用；右键菜单可用；插件架构重构后原有工具功能不受影响

## Phase 3：扩展与跨平台（v0.3）

**目标**：扩充工具库，启动 macOS 兼容验证

**任务清单**

- [ ] 新增工具（按需，优先考虑：PDF 合并、二维码生成、视频压缩、颜色取色器）
- [ ] macOS 构建配置（DMG + Code Signing）
- [ ] macOS 快捷键与托盘适配
- [ ] macOS 深色模式验证
- [ ] 工具标签分类（面板支持按分类过滤）
- [ ] 使用频率统计（自动将高频工具权重排前）
- [ ] 用户体验打磨（动效细节、无障碍访问）

## v1.0 正式版里程碑

**进入条件**：

1. Phase 0–2 全部验收通过
2. 至少 4 款工具稳定可用
3. 自动更新流程在真实设备上验证通过
4. 安装包在 Windows 10 / 11 各版本测试通过（含 EV 代码签名，无 SmartScreen 拦截）
5. 完成用户文档（使用说明 + 常见问题）
6. 本地日志方案在生产环境验证可用
7. **单元测试全部通过，核心处理逻辑覆盖率 ≥ 80%**

## 工具扩展候选列表

以下工具按优先级排序，供后续 Phase 规划参考：
