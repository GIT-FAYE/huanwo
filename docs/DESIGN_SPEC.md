# 唤我（HuanWo）UI/UX 设计规范

**版本：** v0.1  
**日期：** 2026-06-07  
**状态：** 规划中

---

## 1. 设计原则

| 原则 | 说明 |
|---|---|
| 克制 | 只展示当前任务所需信息，不堆砌功能入口 |
| 高效 | 操作路径最短，核心操作不超过 2 次点击 |
| 一致 | 所有工具的 UI 模式统一，用户只需学一次 |
| 可靠 | 操作结果明确反馈，错误状态有明确说明 |

## 3. 快捷面板规范

### 3.1 尺寸与位置

```
面板宽度：520px（固定）
工具窗口默认宽度：560px（由工具配置声明，Phase 2 后迁移至 manifest 的 windowSize 字段）
最大高度：600px（超出可滚动）
位置：屏幕居中
出现动画：scale(0.95→1.0) + opacity(0→1)，时长 120ms，ease-out
消失动画：scale(1.0→0.95) + opacity(1→0)，时长 100ms，ease-in
```

### 3.2 搜索栏

- 高度：52px
- 面板打开后自动聚焦
- 占位符文案："搜索工具…"
- 右侧显示 `Esc` 按键标识
- 输入时实时过滤，搜索结果为空时显示"未找到匹配工具"
- 搜索激活时隐藏"已固定"区块，只显示过滤后的全部工具

### 3.3 工具图标卡片

```
容器尺寸：(面板宽 - 16px) / 4列  ≈ 116px × auto
图标区：44×44px，圆角 12px，彩色背景（由工具配置定义，Phase 2 后迁移至 manifest 的 iconBg/iconColor 字段）
工具名：11px，居中，最多 2 行超出省略
固定标记：图标右上角 8px 紫色圆点 + 边框加深
Hover 状态：背景变为 BG Tertiary
```

### 3.4 右键菜单

已固定工具：
- 打开工具
- 取消固定
- 分隔线
- 工具信息

未固定工具：

## 4. 工具窗口规范

### 4.1 通用布局

所有工具窗口遵循统一的三区结构：

```
┌─────────────────────────────────┐
│  标题栏（工具名 + 关闭按钮）      │  32px
├─────────────────────────────────┤
│                                 │
│  拖放区 / 文件列表               │  ~200px（可变）
│                                 │
├─────────────────────────────────┤
│  参数配置区                      │  ~120px（可变）
├─────────────────────────────────┤
│  操作栏（执行按钮 + 输出路径）    │  56px
└─────────────────────────────────┘
```

### 4.2 拖放区设计

**空状态：**
```
虚线边框（border-dashed，2px）
中央图标 + 文案："拖入文件或点击选择"
支持格式提示（小字）
```

**文件已加载状态：**
```
显示文件列表（文件名 + 大小 + 处理状态图标）
右上角"清空"按钮
底部显示"继续添加"入口
```

**剪贴板检测成功后的 UI 反馈**（P3 补充）：

工具窗口检测到剪贴板含有图片并自动填入列表后，需在拖放区下方显示一行提示：

```
┌──────────────────────────────────────┐
│  ┌────────────────────────────────┐  │
│  │    [图标] 已添加 3 个文件      │  │
│  │   拖入更多文件或点击选择        │  │
│  └────────────────────────────────┘  │
│  📋 检测到剪贴板图片，已自动加入列表   │  ← 紫色小字提示，3秒后自动消失
│  文件列表（添加后展示）               │
├──────────────────────────────────────┤
```

**规范**：
- 提示文字：`📋 检测到剪贴板图片，已自动加入列表`
- 颜色：Primary（`#534AB7`），字号 11px，行高 1.4
- 显示时机：剪贴板自动填入完成后出现
- 自动消失：3 秒后 `opacity` 渐隐（200ms ease-out）
- 若用户手动添加文件（拖拽/选择），此提示不显示

### 4.2.1 GIF 输入提示规范

当文件列表中存在 GIF 文件时，在参数配置区顶部显示提示条：

```
┌─────────────────────────────────────────┐
│  ⚠️ GIF 将转为 WebP 动图（保留动画帧），不支持有损压缩  │  ← 黄色提示条 #FAEEDA 底，#854F0B 文字
├─────────────────────────────────────────┤
│  质量  [━━━━━━━●────] 80              │
│  模式  ○ 智能  ● 有损  ○ 无损        │
├─────────────────────────────────────────┤
```

**规范**：

| 项目 | 定义 |
|---|---|
| 提示条背景 | `Warning Light` `#FAEEDA` |
| 提示条文字 | `Warning` `#854F0B`，12px |
| 提示条边框 | `Warning` `#854F0B`，1px，圆角 6px |
| 消失条件 | 文件列表中无任何 GIF 文件时自动消失 |
| 文件列表标记 | GIF 文件旁显示 `🎞️ GIF` 标签（11px，`Text Tertiary`） |
| Tooltip | 悬停 `🎞️ GIF` 标签时显示"将转为 WebP 动图保留动画" |

**触发时机**：文件列表添加 GIF 文件后立即出现；拖放区、文件选择器、剪贴板输入三种方式均触发。

**设计目标**：用户无需查看文档即知晓 GIF 特殊处理规则，避免处理完后才发现格式变化。


**拖拽悬停状态：**

```
边框变为主色
背景变为 Primary Light
```


### 4.3 执行状态

| 状态 | 执行按钮文案 | 进度展示 |
|---|---|---|
| 待执行 | "开始处理" | — |
| 处理中 | "中止处理"（红色 `#A32D2D`）| 文件列表每行显示进度条，操作栏显示"已完成 X/50" |
| 中止中（完成当前后停止） | "中止中…" + 禁用 | 当前文件进度条继续走完，其余文件标记"已中止" |
| 完成 | "处理完成" + 查看结果 | 每行显示"节省 XX%"或格式标识 |
| 完成（已中止） | "处理已中止" + 再处理 | 已完成项显示结果，未处理项灰色"已中止" |
| 部分失败 | "完成（含错误）" | 失败项红色高亮 + 错误原因 tooltip |

**中止交互细节**：
- 点击"中止处理"后，按钮文案变为"中止中…"，变为禁用状态
- 当前正在处理的文件**继续完成**（不中断当前文件）
- 其余待处理文件状态更新为"已中止"（灰色，不执行）
- 所有文件处理完毕后（含当前完成 + 已中止），进入"完成（已中止）"状态
- "完成（已中止）"状态下，操作栏显示"再处理"（重新处理所有"已中止"项）和"打开输出目录"



### 4.4 输出路径配置

输出策略详见 [TOOLS_SPEC.md](./TOOLS_SPEC.md) 中各工具的「输出策略」章节。

设计层约定：
- 输出路径选择区位于操作栏，与执行按钮并列
- 路径过长时中间省略（`C:\Users\...\output\`）
- 点击路径区可重新选择输出目录

## 5. 深色/浅色模式切换规范

- **触发**：监听系统主题变化，自动切换，无手动开关
- **切换过渡**：`transition: background-color 0.2s, color 0.2s, border-color 0.2s`
- **图标适配**：工具图标彩色背景在深色模式下透明度降至 80%（`opacity: 0.8`）
- **拖放区虚线**：浅色 `rgba(0,0,0,0.15)`，深色 `rgba(255,255,255,0.15)`

## 7. 动效规范

| 场景 | 动效 | 时长 |
|---|---|---|
| 面板弹出 | scale 0.95→1 + opacity 0→1 | 120ms ease-out |
| 面板关闭 | scale 1→0.95 + opacity 1→0 | 100ms ease-in |
| 工具窗口打开 | opacity 0→1 + translateY 4→0 | 150ms ease-out |
| Hover 高亮 | background-color | 80ms ease |
| 处理进度条 | width 渐进 | 实时跟随进度 |
| 主题切换 | background/color/border | 200ms ease |

**原则**：所有动效时长控制在 200ms 以内，不使用弹簧或过于复杂的曲线，保证流畅不拖沓。
