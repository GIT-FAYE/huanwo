## 工具 1：图片压缩（image-compress）

### 1.1 功能定位

对用户选择的图片文件进行有损或无损压缩，在保证可接受画质的前提下尽可能降低文件体积，批量处理。

### 1.2 支持格式

| 输入格式 | 输出格式 | 说明 |
|---|---|---|
| JPEG / JPG | JPEG | 有损压缩，可调质量参数 |
| PNG | PNG | 无损压缩（减少色深 / 去除元数据） |
| WebP | WebP | 有损或无损，可调质量 |
| GIF | GIF | **仅支持转格式（GIF → WebP/APNG，保留动画）**；不支持有损压缩。GIF 输入时 UI 提示"GIF 将转为 WebP 动图或 APNG，保留动画帧" |
| TIFF | JPEG / PNG / WebP | 转换后压缩 |
| AVIF | AVIF | 有损压缩 |
| HEIC | JPEG / WebP | 读取后输出为兼容格式 |

> 底层库：`sharp`（基于 libvips）
> **GIF 提示时机与形式**（1.2 补充）：
> - **参数区提示条**：GIF 文件加入列表后，在参数配置区顶部显示黄色提示条：`⚠️ GIF 将转为 WebP 动图（保留动画帧），不支持有损压缩`，直至文件列表无 GIF 文件时消失
> - **文件列表标记**：GIF 文件旁显示 `🎞️ GIF` 小标签，鼠标悬停 tooltip 显示"将转为 WebP 动图保留动画"
> - 两种提示同时出现，确保用户明确知晓 GIF 处理方式

### 1.3 压缩参数

#### 通用参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| 质量（quality） | 滑块 1–100 | 80 | JPEG/WebP 有损质量，100 为最高质量 |
| 压缩模式 | 单选 | 智能 | 智能 / 有损 / 无损 |

#### JPEG 专属参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| 渐进式 JPEG | 开关 | 关 | 启用后图片从低质到高质渐进加载 |
| 去除 EXIF 元数据 | 开关 | 开 | 减少体积，保护隐私 |

#### PNG 专属参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| 压缩级别 | 滑块 0–9 | 6 | 0=不压缩，9=最大压缩（速度较慢） |
| 色深优化 | 开关 | 开 | 自动降低不必要的位深（如 32bit→24bit） |

#### 尺寸缩放（可选）

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| 缩放 | 开关 | 关 | 启用后可指定最大宽/高度 |
| 最大宽度 | 数字输入 | — | 等比缩放，不超过原始尺寸 |
| 最大高度 | 数字输入 | — | 等比缩放，不超过原始尺寸 |

### 1.4 输出策略（两款工具统一）

| 策略 | 行为 |
|---|---|
| 默认（推荐） | 在源文件同级目录生成 `[原名]_compressed.[原扩展名]`（压缩）或 `[原名].[目标扩展名]`（转格式） |
| 覆盖源文件 | 直接覆盖（执行前显示警告确认） |
| 指定目录 | 输出到用户选择的文件夹，保留原文件名 |

同名冲突策略：追加序号（`img_compressed_1.jpg`）

**压缩/转换结果展示**：每个文件完成后显示"原始 XX KB → 输出后 XX KB（节省 XX%）"

### 1.4.1 剪贴板集成

两款工具均支持以下剪贴板操作：

| 操作 | 触发方式 | 行为 |
|---|---|---|
| 剪贴板作为输入源 | 工具窗口打开/获得焦点时，若剪贴板含有图片 | **追加**到文件列表末尾，不覆盖用户已手动选择的文件 |
| 结果复制到剪贴板 | 处理完成后，文件列表旁"复制"按钮 | 将输出文件写入剪贴板，可直接 Ctrl+V 粘贴到微信/飞书/编辑器 |

### 1.5 UI 布局

```
┌──────────────────────────────────────┐
│  图片压缩                         [×] │
├──────────────────────────────────────┤
│  ┌────────────────────────────────┐  │
│  │   [图标]                       │  │
│  │   拖入图片或点击选择            │  │
│  │   支持 JPG PNG WebP GIF 等     │  │
│  └────────────────────────────────┘  │
│  文件列表（添加文件后展示）            │
│  img001.jpg  2.4MB   → ████ 完成     │
│  img002.png  1.1MB   → ████ 完成     │
├──────────────────────────────────────┤
│  质量  [━━━━━━━━●────] 80            │
│  模式  ○ 智能  ● 有损  ○ 无损        │
│  [✓] 去除 EXIF   [ ] 渐进式 JPEG    │
│  输出  ● 同目录  ○ 覆盖  ○ 指定目录  │
├──────────────────────────────────────┤
│  [清空]          [开始压缩 (3张)]    │
└──────────────────────────────────────┘
```

**处理中操作栏变化：**

```
待执行：  [清空]          [开始压缩 (3张)]
处理中：  [取消]          [中止处理 (已完成 2/50)]   ← 点击后完成当前文件即停止
已完成：  [打开输出目录]  [再压一批]
```

- 处理中"中止处理"按钮为红色（`#A32D2D`），点击后**完成当前正在处理的文件**，随后停止，已处理完成的文件保留结果
- "取消"按钮（处理前）仅在待执行状态出现，清除文件列表

### 1.6 边界条件

- 文件数量上限：单次处理 500 个文件
- 单文件大小上限：500MB（超出提示跳过）
- 处理顺序：按添加顺序逐个处理，并发数 = CPU 核心数（最大 4）
- 压缩后体积反而更大时：输出原文件并提示"已保留原始文件（压缩无收益）"
- **中止处理逻辑**：用户点击"中止处理"后，等待当前文件处理完成并写入结果，随后跳过剩余文件，进入"完成（已中止）"状态。已处理文件的结果保留，不会回滚。

### 1.7 中止处理

| 时机 | 行为 |
|---|---|
| 点击"中止处理" | 完成当前文件 → 停止批处理 → 进入"完成（已中止）"状态 |
| 中止后文件列表 | 已完成项显示正常结果，未处理项显示"已中止"灰色状态 |
| 输出结果 | 已处理完成的文件结果保留，不进行回滚 |
| 摘要文案 | "已处理 12/50 张（用户中止），节省 3.2MB" |

## 工具 2：图片转格式（image-convert）

### 2.1 功能定位

将图片从一种格式批量转换为另一种目标格式，无需关心压缩参数，专注格式转换。

### 2.2 支持转换矩阵

| 输入格式 | 可输出为 |
|---|---|
| JPEG | PNG / WebP / AVIF / TIFF / BMP |
| PNG | JPEG / WebP / AVIF / TIFF / BMP |
| WebP | JPEG / PNG / AVIF / TIFF |
| GIF | WebP / APNG（保留动画） |
| TIFF | JPEG / PNG / WebP |
| HEIC | JPEG / PNG / WebP |
| AVIF | JPEG / PNG / WebP |
| BMP | JPEG / PNG / WebP |
| SVG | PNG（光栅化，指定分辨率） | ⚠️ 需要系统安装 librsvg（Windows 环境默认缺失），v0.1 暂不启用 SVG 输入，推迟至 v0.2 评估 |

> 底层库：`sharp` + `gif2webp`（动图转换）  
> **SVG 依赖说明**：`sharp` 处理 SVG 需要系统层 `librsvg`。Windows 环境默认不安装，v0.1 在 UI 中隐藏 SVG 选项，待 v0.2 评估打包 `librsvg` 或改用其他 SVG 渲染方案后再启用。

### 2.3 转换参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| 目标格式 | 下拉选择 | WebP | 输出格式 |
| 质量 | 滑块 1–100 | 85 | 适用于有损格式（JPEG/WebP/AVIF） |
| 去除元数据 | 开关 | 开 | 去除 EXIF、XMP 等 |
| 背景颜色 | 颜色选择器 | #FFFFFF | 仅在 PNG（含透明）→ JPEG 时生效 |

**SVG 专属参数**

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| 输出分辨率 | 选择（1×/2×/4×/自定义） | 2× | 光栅化倍率 |
| 基准宽度 | 数字输入 | 原 SVG viewBox 宽 | 配合倍率计算最终像素 |

### 2.4 输出策略

与图片压缩工具统一，详见 [1.4 输出策略](#14-输出策略两款工具统一)。

├──────────────────────────────────────┤
│  ┌────────────────────────────────┐  │

│  └────────────────────────────────┘  │
│  文件列表（添加后展示）               │
│  photo.png   1.2MB   PNG → WebP  ✓  │
├──────────────────────────────────────┤

│  输出  ● 同目录  ○ 覆盖  ○ 指定目录  │

**处理中操作栏变化**（与工具1统一）：
- 处理中"开始转换"按钮变为"中止处理"（红色 `#A32D2D`），点击后完成当前文件即停止

### 2.6 边界条件

- 同格式转同格式：允许（等效于去除元数据 + 重新编码）
- 带透明通道转 JPEG：自动填充白色背景，提示用户
- 动图（GIF）转静图格式：仅取第一帧，提示用户
- 超大图片（>8000px 边长）：提示内存风险，需用户确认继续
- **中止处理逻辑**（与工具1统一）：用户点击"中止处理"后，等待当前文件处理完成并写入结果，随后跳过剩余文件，进入"完成（已中止）"状态。已处理文件的结果保留，不会回滚。

## 两款工具共同约定

### 错误处理

| 错误类型 | 处理方式 |
|---|---|
| 文件格式不支持 | 跳过该文件，列表中标红显示原因 |
| 文件无读取权限 | 跳过并提示 |
| 磁盘空间不足 | 中断批处理，显示全局错误提示 |
| 处理异常（库错误） | 跳过该文件，记录错误 log |

### 处理完成行为

1. 托盘图标显示"✓"角标（3 秒后消失）
2. 工具窗口底部展示摘要：共处理 N 张，节省 XX MB，失败 X 张
3. 提供"打开输出目录"快捷按钮
4. 提供"复制结果到剪贴板"按钮（每张输出文件旁均有复制按钮）
5. 处理记录可在会话期间查看（窗口关闭后清除）

### 剪贴板作为输入源（共同支持）

工具窗口打开时，自动检测系统剪贴板：

- 若剪贴板含有图片文件路径（如截图工具 `Ctrl+C` 后）：**追加**到文件列表末尾（不替换已选文件）
- 若剪贴板含有图片二进制数据（如浏览器复制图片）：先写入临时文件，再**追加**到列表末尾
- 检测逻辑在工具窗口 `onFocus` 时触发（用户切回窗口时刷新）

**追加模式说明**：剪贴板填入始终是追加，不覆盖——即用户已手动拖入 3 张图，切走 `Ctrl+C` 复制了另一张再切回，新图追加为第 4 张，不影响前 3 张已选文件。

**去重逻辑（防重复追加）**：

每次 `onFocus` 触发时，检测到剪贴板图片后，先与当前文件列表对比：
- 若文件路径已存在于列表中（针对路径类型）→ 跳过，不追加
- 若剪贴板内容与上次填入时完全相同（针对二进制类型，通过内容 hash 对比）→ 跳过，不追加
- 只有检测到**新增内容**时才执行追加

> 设计目标：用户切走再切回窗口时，若剪贴板未变化，不会产生重复文件条目。

**临时文件管理**：

| 临时文件写入位置 | `os.tmpdir()/huanwo-clipboard/` |
|---|---|
| 文件命名规则 | `clipboard_<timestamp>.<ext>`，如 `clipboard_1749268800000.png` |
| 清理时机 | **工具窗口关闭时**立即删除该窗口产生的临时文件（`BrowserWindow.on('closed')`）；应用退出时清理整个 `huanwo-clipboard/` 目录（`app.on('will-quit')`） |
| 防泄漏保障 | 应用异常崩溃重启时，启动逻辑检测并删除残留临时文件（`app.on('ready')` 时执行清理） |
