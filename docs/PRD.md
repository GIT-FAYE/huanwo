# 唤我（HuanWo）产品需求文档

**版本：** v0.1  
**日期：** 2026-06-07  
**状态：** 规划中

---

## 1. 产品概述

### 1.1 产品定位

「唤我」是一款面向 Windows（后续扩展至 macOS）的桌面小工具合集，通过全局快捷键随时呼出工具面板，让用户在任意场景下快速完成图片处理、文件批量操作等高频琐碎任务，而无需打开浏览器或专用大型软件。

### 1.2 设计理念

- **即呼即用**：快捷键唤起，Esc 退出，零打断主工作流
- **克制而完整**：每个工具只做一件事，但把那件事做好
- **可持续扩展**：架构支持后续插件化（Phase 2 引入），新工具添加便捷，无需大幅修改主程序

### 1.3 目标用户

| 用户群体 | 使用场景 |
|---|---|
| 设计师 / 运营 | 批量压缩上传图片、快速转换格式 |
| 开发者 | 格式转换、文件重命名规则化 |
| 内容创作者 | 图片批处理、素材整理 |
| 普通办公用户 | 文件批量操作、小型格式转换 |

## 2. 核心功能需求

### 2.1 快捷面板（Launcher Panel）

**触发方式**

- 默认快捷键：`Alt + Space`（用户可在设置中自定义）
- 系统托盘右键菜单也可打开面板

**面板布局**

```
┌─────────────────────────────────┐
│  🔍 搜索工具…              [Esc] │
├─────────────────────────────────┤
│  已固定                          │
│  [ 图片压缩 ] [ 图片转格式 ] ...  │
├─────────────────────────────────┤
│  全部工具                        │
│  [ 工具A ] [ 工具B ] [ 工具C ]   │
│  [ 工具D ] [ 工具E ] ...         │
├─────────────────────────────────┤
│  单击打开工具  ·  右键固定/取消   │
└─────────────────────────────────┘
```

**交互规范**

| 操作 | 行为 |
|---|---|
| 打开面板 | `Alt + Space`，面板出现在屏幕中央 |
| 关闭面板 | `Esc` 或点击面板外区域 |
| **再次按下快捷键（Toggle）** | 面板已打开时再次按 `Alt + Space` → **关闭面板**（Toggle 行为，与 Raycast 等同类工具一致） |
| 搜索工具 | 面板打开后直接输入，实时过滤；搜索时隐藏"已固定"区块 |
| 打开工具 | 单击工具图标，面板关闭，工具窗口弹出 |
| 固定工具 | 右键工具图标 → "固定到顶部"；已固定工具右键 → "取消固定" |
| 键盘快捷跳转 | 数字键 `1~9` 对应已固定工具（按固定顺序）；按下后直接打开对应工具 |
| 滚动查看 | 工具数量超出一屏时，全部工具区可滚动 |

**固定工具存储**：持久化到本地配置文件（`config.json`），不随应用重启丢失

### 2.2 工具窗口（Tool Window）

- 每个工具以独立窗口打开，可同时开启多个不同工具
- 窗口支持拖动、最小化，不支持最大化（保持小工具定位）
- 默认窗口尺寸由各工具配置声明（Phase 2 后迁移至 `manifest.json`）
- **文件输入方式**：
  - 拖拽文件/文件夹至工具拖放区
  - 点击拖放区打开文件选择器（支持多选）
  - 两种方式可混合使用（拖拽后可继续追加）
  - **剪贴板直接输入**（详见 2.3）

**窗口关闭行为（重要）**：

> 关闭工具窗口 ≠ 退出应用。应用进程始终驻留在系统托盘，直到用户从托盘菜单选择"退出唤我"。

| 操作 | 行为 |
|---|---|
| 点击工具窗口 `×` | 仅关闭该工具窗口，应用继续驻留托盘，快捷键仍可用 |
| 关闭最后一个工具窗口 | 同上，应用仍驻留托盘，不退出进程 |
| 托盘菜单 → "退出唤我" | 退出整个应用进程，释放全局快捷键 |
| 系统注销 / 关机 | 应用随系统正常退出 |

此行为参考 QQ、微信等常驻托盘工具的交互惯例，确保快捷键随时可用。

**窗口最小化恢复方式（重要）**：

> 工具窗口**不在 Windows 任务栏显示独立条目**（避免与托盘驻留重复），最小化后通过以下方式恢复：

| 恢复方式 | 操作 | 说明 |
|---|---|---|
| 全局快捷键 | `Alt + Space` → 面板 → 点击工具图标 | 适用于记住工具位置的情况 |
| 托盘菜单 | 右键托盘图标 → "打开 [工具名]" | 需要 Phase 2 实现，Phase 0-1 暂不支持 |
| Alt + Tab | 切换至「唤我」工具窗口 | Windows 原生行为，窗口最小化后仍在 Alt+Tab 列表 |
| 重新打开 | 面板 → 点击同一工具 | 若窗口已最小化，自动聚焦并恢复（不新建窗口） |

**Phase 0-1 行为**：工具窗口最小化后，通过 `Alt+Tab` 或重新从面板点击工具图标（检测到已有窗口则 `restore()` 恢复）两种方式恢复。托盘菜单"打开工具"功能推迟至 Phase 2 实现。

### 2.3 剪贴板集成

这是提升工作流效率的关键特性，目标是实现"截图 → 呼出面板 → 直接处理"的零打断流程。

| 功能 | 说明 |
|---|---|
| 剪贴板作为输入源 | 工具窗口打开时自动检测剪贴板，若含有图片则**追加**到文件列表末尾（不覆盖已选文件） |
| 结果复制到剪贴板 | 处理完成后每张输出文件旁提供"复制"按钮，点击后将文件写入剪贴板，可直接 Ctrl+V 粘贴到微信/飞书/Word 等 |

**剪贴板检测时机**：
- 工具窗口获得焦点时（`onFocus`）
- 用户从其他应用 `Ctrl+C` 复制图片后切回「唤我」窗口时自动刷新

### 2.4 系统集成

| 功能 | 说明 | 阶段 | 前置依赖 |
|---|---|---|---|
| 系统托盘驻留 | 应用启动后收入托盘，不占用任务栏 | Phase 0 | 无 |
| 开机自启 | 可选，默认关闭 | Phase 2 | 写入 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 注册表项 |
| 全局快捷键 | 系统级注册，任意前台应用下均可触发 | Phase 0 | 无（冲突时自动降级，见第 4 节设置中心） |
| 深色/浅色模式 | 跟随 Windows 系统主题自动切换 | Phase 0 | Electron `nativeTheme` API |
| 资源管理器右键菜单 | 右键文件 → "唤我 → 压缩图片/转换格式"，无需呼出面板 | Phase 2 | 注册 Windows Shell Extension（`HKCR\*\shell\huanwo`），需管理员权限安装 |
| 处理参数历史记忆 | 记住最近使用的参数模板，下次打开同工具时自动恢复 | Phase 2 | electron-store 持久化（无额外系统依赖） |

### 2.5 自动更新

- 应用内置更新检测机制，启动时检查新版本
- 检测到更新时在托盘图标或设置页显示提示
- 用户确认后后台下载，下次启动时应用更新
- 分发渠道：GitHub Releases（主） + Gitee Releases 镜像（国内备用）
- **代码签名**：v1.0 正式版必须使用 EV 代码签名证书签名，避免 Windows SmartScreen 拦截

## 3. 非功能需求

| 指标 | 目标值 |
|---|---|
| 面板呼出延迟 | < 150ms |
| 安装包体积 | < 80MB（争取 < 40MB） |
| 内存占用（托盘驻留） | < 80MB |
| 图片处理速度 | 单张 1080p 图片压缩 < 2s |
| 系统要求 | Windows 10 1903+ |

## 4. 设置中心

**基础设置**

- 快捷键自定义（含冲突检测与冲突处理）
  - 冲突检测：输入快捷键时实时检测是否与系统或其他应用已注册的热键冲突
  - 冲突处理：检测到冲突 → 托盘弹出气泡提示（\"快捷键 Alt+Space 与其他程序冲突，已禁用\"）→ 设置页标记冲突状态并引导用户自定义新的快捷键
  - 冲突时应用仍可正常启动，只是该快捷键失效，用户可通过托盘右键菜单手动呼出面板
- 开机自启开关
- 语言（中文 / English，v1 仅中文）

**更新设置**

- 自动检查更新开关（默认开启）
- 当前版本号 / 检查更新按钮

**关于**

- 版本信息、开源协议声明

## 5. 插件系统规划（Phase 2 引入）

> Phase 0-1 工具硬编码在主程序中，不抽象插件系统。Phase 2 重构时再引入插件架构。

Phase 2 插件系统目标规范：

每个工具是一个独立插件包，放入 `plugins/` 目录即可自动注册。插件需提供：

1. `manifest.json`：元数据声明（见技术架构文档）
2. 工具 UI 界面
3. 处理逻辑

产品层面的约束：

- 每个插件只做一类任务（单一职责）
- 插件不得访问用户未明确选择的文件路径
- 插件不得进行任何网络请求（工具性质，离线优先）

## 7. 明确不做的事项（Scope Out）

- 不监听特定文件夹自动处理（用户主动触发原则）
- 不提供云同步、账号体系
- 不做任何网络请求（工具插件层面）
- v1 阶段不支持插件市场 / 第三方插件安装
